启动与测试

启动 Zookeeper

启动：

/app/zookeeper-3.4.14/bin/zkServer.sh start

查看状态：

/app/zookeeper-3.4.14/bin/zkServer.sh status

出现如下数据表示成功：

ZooKeeper JMX enabled by default
Using config: /app/zookeeper-3.4.14/bin/../conf/zoo.cfg
Mode: standalone

启动并测试 HBase

start-hbase.sh

输入 jps 查看进程：

1328 QuorumPeerMain
1825 Jps
1549 HRegionServer
1470 HMaster

输入 hbase shell 进入 HBase 命令行，输入list查看已存在的表。

测试 Phoenix

移动到 phoenix 目录：

cd /app/apache-phoenix-5.0.0-HBase-2.0-bin/

连接：

python2 ./bin/sqlline.py 127.0.0.1:2181

查看表：

启动 Kafka

输入命令启动 kafka ， 首先进入kafka 目录 `

cd /app/kafka_2.11-2.4.0/

修改配置文件：

vim config/server.properties

修改 zookeeper.connect 属性：

zookeeper.connect=master:2181

后台启动 kafka

bin/kafka-server-start.sh config/server.properties &

输入 jps 查看是否有 kafka 进程：

1328 QuorumPeerMain
3424 Jps
3033 Kafka
1549 HRegionServer
1470 HMaster

测试 Kafka

创建和查看 topic

创建 Topic：

bin/kafka-topics.sh --create --bootstrap-server localhost:9092 --replication-factor 1 --partitions 1 --topic test

查看创建好的 topic：

bin/kafka-topics.sh --list --bootstrap-server localhost:9092

结果：test

生产和消费数据：

生产数据：

bin/kafka-console-producer.sh --broker-list localhost:9092 --topic test

消费数据：

bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic test --from-beginning

设置Window的hosts文件

在 C:\Windows\System32\drivers\etc 中的 hosts 文件添加如下配置：

192.168.20.100  master
